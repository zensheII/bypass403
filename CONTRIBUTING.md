# Contributing

Thanks for taking the time to contribute.

This project focuses on practical 403 bypass testing techniques for authorized security assessments. Contributions should improve reliability, clarity, or coverage — not add unnecessary complexity.

## How to Contribute

1. Fork the repository
2. Create a new branch (`feature/short-description` or `fix/short-description`)
3. Make your changes
4. Test your changes
5. Submit a Pull Request with a clear explanation

## Contribution Guidelines

- Keep the tool lightweight (standard library only, no external gems)
- Maintain clean and readable Ruby code
- Avoid breaking existing flags or CLI behavior
- If adding new bypass techniques, explain:
  - Why it works
  - In what scenario it may apply
- Do not add features unrelated to 403 testing

## Reporting Issues

When opening an issue, please include:

- Ruby version
- Target setup (lab environment only)
- Command used
- Output received
- Expected behavior

Screenshots or sample output are helpful.

## Responsible Usage

This tool is intended for authorized security testing only.
Do not submit contributions that promote illegal or unethical use.

Small improvements are welcome. Clean ideas are even better.
